git の現在の変更差分を確認して、適切なコミットメッセージを提案し、ユーザーの確認を取ってからコミットしてください。

以下の手順で進めてください：

1. `git status` で変更ファイルを確認する
2. `git diff` で差分の内容を確認する
3. 変更内容を分析して、コミットメッセージの候補を日本語で3つ提案する
   - フォーマット: `<type>: <概要>` （type は feat / fix / docs / style / refactor / chore のいずれか）
   - 例: `feat: localStorage を使ったフォーム入力値の保存機能を追加`
4. ユーザーに選択または修正を求める
5. 承認されたらコミットを実行する
